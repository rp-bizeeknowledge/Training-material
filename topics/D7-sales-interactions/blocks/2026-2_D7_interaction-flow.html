D7 — Section 4

Interaction
Flow

Stay real, stay ready. No scripts.
The Four-Step Flow
1

Greeting

Start the call by introducing yourself and the company. Then, warmly ask how you can assist the customer.

"Thank you for calling Bizee. My name is [Name]. How can I help you today?"

2

Query

Figure out what the customer needs by listening actively and asking probing questions. Authenticate only if needed (see authentication rules below).

3

Support

After the customer expresses their concern and has been authenticated, address the issue clearly and helpfully. Before ending the call, offer any additional support or information they may need.

"I'm glad I was able to assist. Is there anything else I can help with?"

4

Closing

End with a friendly goodbye, then document the call — including any actions taken and call details — in an internal note in Zendesk and comment in DB when appropriate.

"Thank you for calling Bizee. Have a great day!"

2.1 — Authentication

We protect customer data by verifying account holders before sharing or changing account information. Authenticate when you cannot pull up the account using Zendesk user details, and the customer wants to access account-specific information, make changes, or cancel a subscription.

Based on information sensitivity, some questions require authentication and some do not:

✓ No Auth Required

  • Public information questions
  • "How long does it take to file an LLC?"
  • "How often are Annual Reports required in Oklahoma?"
  • "I need to start a new Corporation"

⚠ Authentication Required

  • Account-specific or personal information
  • "Can I get my EIN?"
  • "Please cancel my service."
  • "I need to update my company's membership"
2.1.1 — How to Authenticate

Step 1: Review the customer's billing information in the database to determine what to ask. If no payment method is on file, adjust your questions.

Step 2: Use the fallback chain below — start at A and work down only if needed:

A

Customer's name + last 4 digits of card on file

Name must match the account. If no card on file, move to billing address.

B

Full billing address

Street, city, or zip code on file.

C

Last 4 digits of SSN

Only if EIN is included in the order. Note: Apple Pay uses a temporary card number — the last 4 digits in DB will not match the customer's real card; do not use them for authentication.

D

Contact from the information on file / Photo ID

Ask them to call or email us from the contact information on file. As a last resort, send an email to the preferred address asking for a photo of their ID.

Step 3 — If the Caller Is Not a Verified Contact: Ask to speak directly with the verified contact or offer to transfer the call to them. If they cannot authenticate, do not provide account-specific help.

🚫

Key rule: Do not tell customers to "reach out from the email or phone on file." Always follow the full authentication process.

2.1.2 — Roles and Permissions
Contact
Can receive information and request changes.
Client
Can receive information but can only make changes through their dashboard.
Owner (green check in DB)
Can receive information but can only make changes through their dashboard.
2.1.3 — Special Scenarios

If a verified contact is present with the caller, authenticate the contact directly.

If the sole member of a company managed by a commercial client wants ownership: send an email requesting their ID → collect information to add them as a user in the database → escalate using Zendesk (by selecting your Supervisor in the dropdown) and mark the ticket as pending.
