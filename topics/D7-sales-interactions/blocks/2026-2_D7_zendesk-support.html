What It Is

At Bizee, Zendesk is used as the centralized system to manage all customer interactions, including phone calls and emails. Each time an existing or prospective customer contacts us through any of these channels, a Support Ticket is automatically created and assigned a unique ticket number for tracking and reference purposes.

2.1 — Glossary
Ticket
A single record in Zendesk that represents a customer request, question, or issue, including all related messages, updates, and actions taken to resolve it.
User
Any person registered in Zendesk who interacts with the system (End User / Staff Member).
Assignee
The agent or group currently responsible for handling and resolving a ticket.
Requester
The user who originally submitted the ticket. This is typically the customer requesting assistance.
CC (Carbon Copy)
A feature that allows additional users to be included on a ticket so they can receive public replies and updates.
Group
A collection of agents organized by role, expertise, or function, used to route and manage tickets efficiently.
Escalation Box
A designated field used to flag and forward cases that require special handling, input, or review by another team.
Internal Note
A private note visible only to staff, used to document internal context, actions taken, or instructions that should not be shared with the customer.
Public Reply
A response sent through the ticket that is visible to the customer and any users included in CC.
CSAT
Customer Satisfaction. A metric used to measure customer satisfaction based on a survey sent after a ticket is resolved.
2.3 — Merging Guidelines: Tickets

✅ Do

  • Merge tickets when multiple tickets exist for the same issue. Consolidate them into the ticket you will actively respond to, ensuring all customer questions are addressed in a single thread.
  • Merge new tickets into your existing open or pending ticket when they relate to the same request. This helps prevent duplicate work and avoids unnecessary or duplicate escalations.

❌ Don't

  • Merge tickets into solved tickets, unless the ticket documents a callback or action that occurred before the issue was resolved and needs to be referenced.
  • Merge a ticket into another ticket that has not been reviewed or worked on.

Option A — Merging one ticket into yours: Click the 3 dots in the top right corner → "Merge into another ticket" → enter the ticket number to merge into → confirm → deselect "User can see this comment" checkboxes if applicable.

Option B — Merging various tickets from your views: Select the tickets → click "merge" at the bottom → type in the ticket number or select from the list → click merge.

Option C — Merging various tickets from the same user: Click the user's name tab → select the tickets to merge (not the one you're merging into) → click the down arrow on the black box → "Merge tickets into another ticket..." → select the target ticket → click merge.

⚠️

BE CAREFUL: ONCE A TICKET IS MERGED, NO ONE CAN UNMERGE IT.

2.3.2 — Merging Guidelines: Users

✅ Do

  • Merge users when a customer already has an account but the email address or phone number they are using is not linked to their existing orders.
  • Authenticate the customer before merging any user records. Identity verification must always be completed first.
  • Offer to merge the information proactively to keep the customer's account consolidated.
  • Obtain clear and explicit customer consent before proceeding. Use clear consent language: "Would you like me to link this phone number (or email) to your account so we can keep everything in one place moving forward?"

❌ Don't

  • Merge users without customer consent, even if the information appears to belong to the same person.
  • Merge users before authentication has been completed.
  • Assume consent based on context or prior interactions.
  • Merge user records unnecessarily if the contact information is already correctly linked.

How to merge users: Click on the customer name → click the down arrow in the upper-right corner → "Merge into another user" → enter the email address or customer's name → click merge.

2.4 — Annotation Guidelines

Always add clear, concise internal notes to every ticket you modify. Include the following:

Every Internal Note Should Include

  • Order number (if applicable)
  • Relevant interaction details
  • Actions taken
  • Next steps (if applicable)
