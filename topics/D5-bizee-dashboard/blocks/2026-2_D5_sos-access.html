D5 — Section 2

SOS Access Requests
by State

South Dakota · North Dakota · Arizona · Utah · California
Overview

We support customer access requests for certain Secretary of State (SOS) online business accounts. In these states, customers can request credentials to manage a company we formed directly through the SOS website. Our procedure helps guide them through the process while maintaining security and compliance standards.

📌

For states not listed below, customers should be instructed to contact their Secretary of State office directly for instructions on accessing their business account.

State Procedures

2.1 South Dakota

  1. Let the customer know that they first need to request access from the South Dakota Secretary of State's website so we can proceed.
  2. Gather required information by sending the customer an email, asking them to provide:
    Required Info
    • Full name used when submitting the SOS access request
    • Email address used in the SOS access request
  3. Once you receive the above information, send: "Thank you for sharing this information. I'll go ahead and escalate this request to the correct department in order to further assist you. As soon as I have an update regarding this matter, I'll be sure to notify you."
  4. Escalate to your supervisor, including in the internal note:
    Escalation Info
    • Order number and company name
    • Requester's name
    • Email used
    • Brief description of the request

Once confirmed that access has been granted, notify the customer that they now have access to their company information on the SOS website.

2.2 North Dakota

  1. Let the customer know that they first need to create an account on the North Dakota Secretary of State's website using their name and email address. Once the account is created, they can log in and click on the request access option.
  2. Gather required information:
    Required Info
    • Full name used when creating the SOS account
    • Email address used when creating the SOS account
  3. Once received, send the same escalation message as above and escalate to your supervisor with the same escalation info.

Once confirmed, notify the customer that access has been granted.

2.3 Arizona

  1. Let the customer know that they need to create an account on the Arizona Secretary of State's website using their name and email address first so we can grant access.
  2. Gather required information:
    Required Info
    • Full name used when creating the SOS account
    • Email address used when creating the SOS request
  3. Once received, send the same escalation message and escalate to your supervisor with the same escalation info.

Once confirmed, notify the customer that access has been granted.

2.4 Utah

  1. Let the customer know that they need to create a Utah-ID account within the SOS website before we can grant access.
  2. Gather required information:
    Required Info
    • Requester's name
    • Email address used to create the Utah-ID account
  3. Once received, send the same escalation message and escalate to your supervisor with the same escalation info.

Once confirmed, notify the customer that access has been granted.

2.5 California

  1. Let the customer know that they first need to create an account on the California Secretary of State's website using their name and email address. Once the account is created, they can log in and click on the request access option.
  2. Gather required information:
    Required Info
    • Full name used when creating the SOS account
    • Email address used when creating the SOS request
  3. Once received, send the standard escalation message and escalate to your supervisor with the standard escalation info.

Once confirmed, notify the customer that access has been granted.

If Bizee is requesting access to the California SOS on behalf of the customer:

  1. Explain that this is part of a new CA SOS system request.
  2. The customer must call the state using the number provided in the issue email and authorize access for Bizee.
  3. The state will provide a PIN or other confirmation.
  4. The customer must send this PIN back to us by replying to the issue email (or support@bizee.com).
  5. Escalate this information to the Order Issues department.
  6. No further action is needed from the agent after escalation.
