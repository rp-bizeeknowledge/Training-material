TOOL USAGE & PROCEDURES · Topic 04

Interaction Flow

Everything you need to know for an effective interaction with customers

Stay real, stay ready. No scripts.

Greeting

Start the call by introducing yourself and the company. Then, warmly ask how you can assist the customer.

Query

Figure out what the customer needs by listening actively and asking probing questions. And authenticate only if needed.

Authentication

We protect customer data by verifying account holders before sharing or changing account information. Authenticate when you cannot pull up the account using Zendesk user details, and the customer wants to:

  • Access account-specific information
  • Make changes
  • Cancel a subscription

Based on information sensitivity, some questions require authentication and some do not:

Public information - No Authentication Required

  • "How long does it take to file an LLC?"
  • "How often are Annual Reports required in Oklahoma?"
  • "I need to start a new Corporation"

Account-Specific or Personal Information Questions - Authentication Required

  • "Can I get my EIN?"
  • "Please cancel my service."
  • "I need to update my company's membership"
How to Authenticate

Step 1 • Determine if Authentication is Needed

✅

No Authentication Required


Callers and email senders are considered authenticated when they contact us using the phone number or email address registered on file.

No additional verification needed if this matches our records and pulls the account up in DB/Bizee Tech.

❌

Authentication Required


If the email or phone number shown in Zendesk is not registered in DB/Bizee Tech, you must verify the individual's identity.

Additional steps are required if the account was not located.

Step 2 • Locate the Account

Use the following information in order when trying to pull up an account in DB/Bizee Tech.

A
Order number
B
First and last name of the point of contact
C
Company name
D
DB link on Zendesk (if shown)

Step 3 • Authenticate (Fallback Chain)

Work through each option in order. Only move to the next if the customer cannot provide the current one.

A

Phone number / Email on file

Check if the phone number or email on file matches a profile in the client tab on DB.

If unable to verify — continue to B ↓
B

Payment details

Ask for any of the following:

Last 4 digits of primary card Last 4 of temp. Apple Pay card (via dashboard) PayPal email
If unable to verify — continue to C ↓
C

Billing address

Ask for any of the following:

Street + city Billing zip code
If unable to verify — continue to D ↓
D

Last transaction

Ask for the date and amount of the last charge processed for the account (transaction history).

⚠

If still unable to authenticate, direct to Account Recovery

Account Recovery

If the caller cannot provide any authentication information, ask to speak directly with the verified contact or offer to transfer the call to them. You can also request them to reach out from the email or phone number on file so you can grant access and update the account if required.

If they cannot authenticate, do not provide account-specific help if authentication fails.

Roles and Permissions
Contact information shown in the Contact tab in Bizee Tech
Client information shown in the Client tab in Bizee Tech
1
Contact

Can receive information and request changes.

2
Owner

Can receive information and make any changes.

3
Client

Can receive information but can only make changes through their dashboard.

Special Scenarios
  • If a verified contact is present with the caller, do not ask them to reach out from the email or phone number on file. Instead, follow the full authentication process.
  • If the sole member of a company managed by a commercial client wants ownership:
    • Send an email requesting their ID
    • Collect information to add them as a user in the database
    • Escalate using Zendesk (by selecting your Supervisor in the dropdown) and mark the ticket as pending
Support

After the customer expresses their concern, and has been authenticated, address the issue clearly and helpfully. Before ending the call, offer any additional support or information they may need. For example:

I'm glad I was able to assist. Is there anything else I can help with?

Closing

End with a friendly goodbye. For example:

Thank you for calling Bizee. Have a great day!

Then document the call, writing down any actions taken and call details in an internal note in Zendesk and comment in DB/Bizee Tech when appropriate.
